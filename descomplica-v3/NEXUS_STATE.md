### 🧠 Implementações Core (The 4-Layer Sieve)
1.  **Layer 1: Context Hashing:** Cache de cenários no Supabase (`context_cache`). Latência zero para buscas idênticas.
2.  **Layer 2: Strict Tribunal (Index-Based):** Auditoria técnica que mapeia celulares por ID numérico, garantindo que cada veredito corresponda 100% ao modelo correto, independente do tamanho do nome.
3.  **Layer 3: Validation Sieve:** Filtro que rejeita termos triviais e exige profundidade técnica (especificações de hardware vs marketing).
4.  **Layer 4: Atomic Persistence:** Injeção autônoma de novos dispositivos no banco com sanitização radical e vereditos salvos como descrição.
